# 代码风格规范

- 面向项目的代码层规则，聚焦命名、格式、导入、注释、类组织和方法设计。

---

## 概览

代码风格规范解决的是“代码看起来是否稳定、读起来是否顺畅、改起来是否不容易出事故”。

这里不讨论事务、分层、数据库和安全边界，那些属于工程层问题。本文只管代码本身的可读性、统一性和基本表达质量。

## 格式规范

> 对于缩进、行宽、换行、大括号、空行和表达式换行规则的定义。

> 对于是否允许单行分支、省略大括号或混用多种排版风格的约束说明。
